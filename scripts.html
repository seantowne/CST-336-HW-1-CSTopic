Setting Permisions on a file

There are a lot of great tutorials for how to change file permisions. Here is one . For our purposes we only need chmod. Lets make a file called myScript and make it excecutable.. 

                        
                            $ touch myScript 
                            $ chmod +x myScript

Putting Commands Into a File

There are much better ways of editing a script file, namely, text editors, but for the purposes of this tutorial we will just use echo. 

                        
                            $ echo "#! /bin/bash" >> myScript 
                            $ echo "echo Hello, World" >> myScript
                            $ cat myScript 
                            #! /bin/bash
                            echo Hello, World

Run The Script!

We just made a script! Lets run it using the './scriptName' syntax.. 

                        
                            $ ./myScript 
                            Hello, World

Our script simply prints out a friendly message, because, thats what we told it to do! That '#! /bin/bash" thing specifies myScript as a bash script. That line where we used the word 'echo' twice was echo-ing in the echo command. Meta huh.

Limitations

Our script will only run from within the directory it is in. In other words, if you wanted to run our script from outside its directory, you would need to specify either the absolute path to it, or the relative path to it from your working directory. Ill show you how to get around that in the path tab of this site. 

                        
                            $ pwd
                            /tutorial
                            $ ls
                            fileCopy.txt  file.txt  myScript  test
                            $ cd test/
                            $ ./myScript
                            bash: ./myScript: No such file or director
